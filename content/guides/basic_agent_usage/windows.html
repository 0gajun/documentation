--- title: Basic Agent Usage for Windows ---

Overview

This guide will outline the basic functionality of the Datadog agent. If you haven't installed the agent yet, instructions can be found here.
The agent can be installed with a CLI installer (recommended) or a GUI installer. It will be installed as a Windows service.

Starting and Stopping the Agent

The execution of the agent is controlled by a Windows service. To start/stop/restart the agent, go to the Services panel. To open it, execute:
<%= console 'services.msc' %> Look for the "Datadog Agent" service. A right click on it gives the different actions: Start, Stop, Restart.

Status and Information

To check if the agent is running, check if the service status in the Services panel is "Started". A process called "ddagent.exe" should also exist in the Task Manager. To receive more information about the agent's state: <%= console 'http://localhost:17125/status' %>

Configuration

The configuration file location depends on the version of Windows. For Windows Server 2003, XP or older
  • Agent configuration:
    C:\Documents and Settings\All Users\Application Data\Datadog\datadog.conf
  • Integration configuration:
    C:\Documents and Settings\All Users\Application Data\Datadog\conf.d\
For Windows Server 2008, Vista and newer
  • Agent configuration:
    C:\ProgramData\Datadog\datadog.conf
  • Integration configuration:
    C:\ProgramData\Datadog\conf.d\

Troubleshooting

First, check if the agent is running in the Services panel and in the Task Manager. Next, try running the status page to see the state of the agent. Logs for the subsystems are available in the Event Viewer.
