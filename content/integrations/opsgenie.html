Overview

Create alerts using @opsgenie:
  • From your event stream
  • By taking a snapshot
  • When a metric alert is triggered

Configuration

Create a DataDog integration in OpsGenie

  1. Log in to your OpsGenie account.
  2. Go to the OpsGenie Integrations page.
  3. Click on the DataDog Tile.
  4. Enter your Datadog API key in the dedicated field. You can find your key here. It should look like this: 7985646abcdefgh79r0q643q
  5. Choose the recipients in OpsGenie and set up your filters.
  6. Save the configuration.
  7. Copy the red key. You will use this in DataDog.
  8. Add more DataDog integrations on OpsGenie by going to the OpsGenie Integrations page and repeating the steps above.

List the integration(s) you made in OpsGenie in DataDog

  1. In Datadog, select the 'OpsGenie' tile on Account Integrations.
  2. In the Configuration tab, paste the key(s) provided for each DataDog integration (created in OpsGenie) in the "DataDog Integration Key" field, and choose a unique "DataDog Integration Name".

How to Use DataDog and OpsGenie Together

Create, acknowledge and close an OpsGenie alert from DataDog

  • You can create an OpsGenie alert by putting @opsgenie-service_name or @opsgenie in the “Say What’s Happening” field, section 5, in the Edit Metric Alert. When this alert is triggered in Datadog, an alert will be sent to the recipients in your OpsGenie service.
  • Someone can acknowledge or close the alert from Datadog with the @opsgenie-acknowledge or @opsgenie-close mentions.

Receive, acknowledge and close an alert created by OpsGenie in DataDog

  • When an alert set-up in OpsGenie is triggered, the reverse will happen, an event will be created in Datadog. The tags and description field from OpsGenie will be carried over to DataDog and the alert will initially be in the error state.
  • An alert can be acknowledged and closed from Opsgenie. When this happens, the acknowledged alert in Datadog will be updated with the username of the person who closed this alert.
