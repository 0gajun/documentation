Overview

This section details Datadog's HTTP API. It makes it easy to get data in and out of Datadog. The Datadog API uses resource-oriented URLs, uses status codes to indicate the success or failure of requests and returns JSON from all requests. Let's dive in and see how it works.

There are many libraries that make it easier to interact with the Datadog API.

Authentication

All Datadog API calls must be authenticated. Requests that write data require reporting access and must be authenticated with a valid API key. API keys is passed as a query parameter: /api/v1/some/service?api_key=your_api_key Requests that read data require full access and must be authenticated with a valid API key and an application key: /api/v1/some/service?api_key=your_api_key&application_key=your_application_key

You can manage your API and application keys on the account settings page.
