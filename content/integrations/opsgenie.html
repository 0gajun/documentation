Overview

Create alerts using @opsgenie:

  • From your event stream
  • By taking a snapshot
  • When a metric alert is triggered

Configuration

Create a DataDog integration on OpsGenie

  • Go to the OpsGenie Integrations page
  • Click on the DataDog Tile
  • Enter your Datadog API key in the dedicated field. You can find your DataDog API key by going to the Integration API page. It should look something like this: 7985646abcdefgh79r0q643q
  • Choose the recipients in OpsGenie and set up your filters
  • Save the configuration
  • Copy the red key. You will use this in DataDog
  • Add more DataDog integrations on OpsGenie by going to the OpsGenie Integrations page and repeating steps 1-7.

Enable the OpsGenie integration in DataDog

  • Select the 'OpsGenie' tile on the Account Integrations page in Datadog.
  • In the Configuration tab, paste the key(s) provided for each DataDog integration (created in OpsGenie) in the DataDog Integration Key field, and choose a unique DataDog Integration Name for it.

How to Use DataDog and OpsGenie Together

Create and acknowledge an OpsGenie alert from DataDog

You can create a Metric Alert on Datadog and put @opsgenie-service_name or @opsgenie in section 5, the “Say What’s Happening” field. When this DataDog alert is triggered, an event will appear in your event stream and an alert will be created and sent in OpsGenie to the recipients you set-up in your OpsGenie service. Someone can acknowledge the alert on Datadog with the @opsgenie-acknowledge mention to say they are working on the issue.

Create, acknowledge and close an alert from OpsGenie in DataDog

Similarly, when an alert set-up in OpsGenie is triggered, the reverse will happen, an event will be created in Datadog. The tags and description field from OpsGenie will be carried over to DataDog in the initial error state, which is denoted in red. An alert can be acknowledged and closed from Opsgenie. In DataDog, the username of the person who acknowledged this alert will appear in the update and the event state will be updated using the following color code concept:
  • red is an error
  • yellow is a warning
  • grey is info
  • green is success
In this case it will be green as the alert is being closed.
